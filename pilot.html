Overview

The Hardseal Design Partner Engagement is a focused 2–4 week pilot designed to integrate the Hardseal compliance evidence engine into an RPO's existing CMMC assessment workflow.

The engagement validates Hardseal's offline-first, OSCAL-native evidence collection against a single real (or representative) CMMC Level 2 enclave. Goal: prove the tool saves time and produces assessor-ready artifacts before broader rollout.

Scope
  • One defined client environment (1 enclave)
  • Full NIST 800-171 Rev 2 evidence collection across all 14 control families (110 controls)
  • OSCAL v1.1.2 output with Ed25519 cryptographic signing
  • Baseline → Current → Diff → OSCAL export → HTML report pipeline
  • Air-gapped or connected — works in both
What We Each Bring

RPO Provides

  • Access to one client enclave (or representative test environment)
  • Subject matter expertise on current assessment workflow
  • Feedback on evidence quality and assessor readiness
  • 2–3 hours per week for check-ins

Hardseal Provides

  • Full Hardseal deployment (zero dependencies, pure Python stdlib)
  • Onboarding and configuration support
  • Custom control family mapping if needed
  • Ongoing iteration based on your feedback
  • All OSCAL output files and HTML reports
Timeline
Week 1: Onboarding
Deploy Hardseal, configure for enclave, run baseline snapshot
Week 2: Evidence Collection
Full 110-control evidence run, OSCAL export, initial review
Week 3: Refinement
RPO feedback integration, evidence quality tuning, assessor readiness check
Week 4: Deliverables
Final OSCAL package, HTML report, pilot summary, go/no-go decision
Deliverables
  • Validated OSCAL v1.1.2 component definition (machine-readable, FedRAMP-format compatible)
  • Ed25519-signed evidence chain (tamper-evident, reproducible)
  • HTML assessment report (human-readable, shareable with clients)
  • Snapshot diff analysis (baseline vs current state)
  • Pilot summary with time-saved metrics vs manual collection
